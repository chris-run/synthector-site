What it does

Synthector sits between sources (transcripts/documents/chats) and downstream AI/analytics. It deterministically transforms content, verifies the transformed output, and emits metadata-only evidence.

Deterministic redaction

Deterministic span detection and replacement using regex/dictionaries plus guarded entity detection.

  • Stable outputs for identical inputs + ruleset
  • Normalized placeholders ([PERSON_1], [EMAIL])
  • Policy/ruleset identity carried as metadata

Posthoc leak-gate

A verification pass that validates the anonymized output and can fail closed (block) when required.

  • Posthoc leak check runs on anonymized output only.
  • Configurable: pass / block / quarantine (metadata)
  • Predictable “stop-the-line” semantics

Attestation + provenance

Metadata-only provenance plus optional signed attestation (JWS) for audit and downstream verification.

  • Ruleset hash + policy ID
  • Gate result + counters
  • No content included in evidence

Tenant isolation + aggregates

Tenant-scoped policies/rulesets and safe telemetry modes that retain only numeric/categorical aggregates.

billing_only health_kmin off

Optional synthetic profiles

Optional tenant-scoped synthetic replacements. Output-only; no retention of originals or synthetic artifacts.

  • Non-retained synthetic outputs
  • Useful for QA, demos, partner workflows

How it works

A deterministic pipeline with an explicit zero-retention boundary: Input → Detect → Redact → Leak-check → Attestation → Output.

  1. Input — text enters from transcripts, chats, notes, or document pipelines.
  2. Detect — deterministic detectors identify sensitive spans (regex/dict + guarded NER).
  3. Redact — spans are replaced with normalized placeholders (or optional tenant-scoped synthetic output).
  4. Leak-check — posthoc gate validates anonymized output; can fail closed.
  5. Attestation — emit metadata-only provenance + optional signed JWS token.
  6. Output — return redacted text + gate result + evidence metadata to your pipeline.
Zero-retention boundary

Only in-memory processing. No raw text storage. Synthector processes content in-memory and does not retain customer content artifacts.

Why deterministic matters

Determinism provides reproducibility, auditability, and predictable failure modes—reducing “surprise behavior” at the privacy boundary.

  • Reproducibility: same input + same ruleset ⇒ same output. Enables replay and forensics.
  • Auditability: policy + ruleset identity (hash) attaches change control to every output.
  • Predictable failure modes: leak-gate can fail closed and stop the line explicitly.
  • Less LLM surprise behavior: privacy enforcement is not a generative rewrite step.

Technical overview

A one-page technical view for security/platform/compliance teams evaluating Synthector as a deterministic privacy boundary.

Architecture

Synthector sits in the ingestion path. It accepts content, applies a deterministic policy/ruleset, runs a leak-gate on anonymized output, and returns transformed output plus evidence metadata.

  • Transform boundary: detect → redact → leak-check
  • Evidence channel: provenance + optional JWS attestation
  • Telemetry channel: aggregate-only counters (mode dependent)

Policy & rulesets

Policies specify what must be removed and what the gate must enforce. Rulesets implement that policy with deterministic detectors. Ruleset identity is carried as a hash for audit/replay.

  • Explicit policy_id + versioning
  • Ruleset hash to make change-control explicit
  • Tenant-scoped separation

Leak-gate semantics

The gate verifies only anonymized output, reducing leakage pathways and enabling explicit stop-the-line behavior.

  • Posthoc leak check runs on anonymized output only.
  • Actions: pass / block / quarantine (metadata-only)
  • Fail-closed configurable

Retention & telemetry

Raw and redacted artifacts are not persisted. Telemetry is aggregate-only when enabled.

  • Zero retention by default.
  • Modes: billing_only, health_kmin, off
  • Provenance is metadata-only and TTL-limited

FAQ

Short, technical defaults.

Do you store transcripts, raw text, or redacted text?
No. Synthector processes content in-memory and does not retain customer content artifacts. Raw and redacted text/audio are never stored. Optional telemetry is aggregate-only (numeric/categorical) and mode-controlled.
What does “leak-gate” mean?
It’s a verification pass that checks the anonymized output for residual disallowed patterns and can fail closed (block). Posthoc leak check runs on anonymized output only.
Do you claim SOC 2 / ISO certifications?
No certification claims are made on this site. The goal is deterministic, inspectable controls that support your compliance program.
